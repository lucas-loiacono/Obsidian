# Files, Exceptions y JSON en Python

## Reading from a File

python

```python
from pathlib import Path  # ya viene incluida en python, no hay que instalar nada

# crea una instancia de Path con el nombre del archivo
# el archivo tiene que estar en el mismo directorio que el script
path = Path('pi_digits.txt')

contents = path.read_text()  # lee el contenido y lo guarda como string
print(contents)
```

### Eliminar espacios en blanco sobrantes

python

```python
from pathlib import Path

path = Path('pi_digits.txt')
contents = path.read_text()
contents = contents.rstrip()  # elimina espacios en blanco al final del string
print(contents)
```

> 💡 Los **paths relativos** se usan para archivos en el mismo directorio (`'data.txt'`) o en subdirectorios (`'files/data.txt'`).

## Trabajar con el contenido de un archivo

python

```python
from pathlib import Path

path = Path('pi_digits.txt')
contents = path.read_text()
lines = contents.splitlines()

pi_string = ''  # variable vacia para acumular el contenido sin saltos de linea
for line in lines:
    pi_string += line.strip()  # strip() elimina espacios al principio y al final de cada linea
```

### Slicing en strings grandes

python

```python
from pathlib import Path

path = Path('pi_digits.txt')
contents = path.read_text()
lines = contents.splitlines()

pi_string = ''
for line in lines:
    pi_string += line.strip()

# [:52] muestra solo los primeros 52 caracteres
print(f"pi_string: {pi_string[:52]}")
print(len(pi_string))  # cuenta el total de caracteres
```

## Writing to a File

python

```python
from pathlib import Path

# si el archivo no existe, se crea automaticamente
# si ya existe, se sobrescribe — se pierde el contenido anterior
```

python


## Exceptions

### try-except básico

python

```python
# sin try-except el programa se detiene con un error
# con try-except podemos manejarlo de forma elegante
try:
    print(5/0)
```

### FileNotFoundError

python

```python
from pathlib import Path

path = Path('alice.txt')
try:
    # encoding='utf-8' evita problemas de codificacion
    contents = path.read_text(encoding='utf-8')
except FileNotFoundError:
    print("Sorry, the file alice.txt does not exist.")
```

## Storing Data: JSON

### json.dumps() — guardar datos en un archivo JSON

`write_text()` solo puede escribir strings, por eso hay que convertir primero con `json.dumps()`.

python

```python
from pathlib import Path
import json

numbers = [2, 3, 5, 7, 11, 13]
path = Path('numbers.json')

# dumps() convierte el objeto python a un string en formato JSON
contents = json.dumps(numbers)
path.write_text(contents)
```

### json.loads() — leer datos de un archivo JSON

python

```python
from pathlib import Path
import json

path = Path('numbers.json')
contents = path.read_text()

# loads() convierte el string JSON de vuelta a un objeto python
numbers = json.loads(contents)
print(numbers)
```

## Guardar y leer datos del usuario

### Guardar el nombre

python

```python
from pathlib import Path
import json

username = input("What is your name? ")
path = Path('username.json')

# convierte el string a formato JSON y lo guarda
contents = json.dumps(username)
path.write_text(contents)
print(f"We'll remember you when you come back, {username}!")
```

### Leer el nombre guardado

python

```python
from pathlib import Path
import json

path = Path('username.json')
contents = path.read_text()

# convierte el string JSON de vuelta al nombre del usuario
username = json.loads(contents)
print(f"Welcome back, {username}!")
```

### El Problema Principal

El disco duro de tu computadora y el método `write_text()` de Python solo entienden de **texto plano** (cadenas de caracteres o _strings_). No saben qué es una "lista de Python" o un "diccionario".

Si intentas guardar una lista directamente, Python te dará un error. Por eso necesitamos un "idioma intermedio", y ahí es donde entra **JSON** (JavaScript Object Notation).

### 1. Guardar Datos (Serializar)

Es el proceso de convertir un objeto "vivo" de Python en texto plano para poder guardarlo.

- **`json.dumps(dato)`:** Toma tu lista de números (o el nombre de usuario) y lo "traduce" a una cadena de texto con formato JSON.
    
- **`path.write_text(contents)`:** Toma ese texto y lo inyecta en el archivo `.json`.
    

### 2. Leer Datos (Deserializar)

Es el proceso inverso: tomar texto plano de un archivo y "darle vida" de nuevo dentro de Python.

- **`path.read_text()`:** Va al archivo `.json` y extrae todo su contenido como una simple cadena de texto cruda.
    
- **`json.loads(contents)`:** Lee ese texto, reconoce la estructura JSON, y lo "reconstruye" como un objeto real de Python (volviendo a ser una lista o un string que puedes usar en tu programa).
