¡No exactamente! Ese es un error súper común al empezar a usar estos métodos (asumiendo que te referís a Python).

Los métodos `lstrip()` y `rstrip()` **no eliminan solo la primera ocurrencia ni tampoco eliminan una "palabra" o prefijo exacto**. Lo que hacen es eliminar **todas** las ocurrencias de los caracteres que le pases, empezando desde el extremo (izquierdo o derecho), **hasta que se chocan con un carácter que no está en esa lista**.

Aquí tenés cómo funcionan realmente:

### Cómo funcionan `lstrip` y `rstrip`

Tratan el argumento que les pasás como un _grupo de caracteres sueltos_, no como un bloque.

Python

```
texto = "rrrtttexto_finaltt"

# lstrip elimina todas las 'r' y 't' desde la izquierda hasta encontrar la 'e'
print(texto.lstrip("rt")) 
# Resultado: "exto_finaltt"

# rstrip elimina todas las 't' desde la derecha hasta encontrar la 'l'
print(texto.rstrip("t"))  
# Resultado: "rrrtttexto_final"
```

### La alternativa: Si querés eliminar algo una sola vez

Si tu objetivo era eliminar un prefijo o un sufijo específico una sola vez (como una palabra o un patrón exacto), desde Python 3.9 existen métodos diseñados exactamente para eso: **`removeprefix()`** y **`removesuffix()`**.

Python

``` python
texto = "hola_hola_mundo"

# removeprefix saca solo el primer bloque exacto
print(texto.removeprefix("hola_")) 
# Resultado: "hola_mundo"
```
